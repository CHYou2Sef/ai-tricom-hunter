Gemma 4 via Ollama — transition plan
Once Gemma 4 lands in the Ollama registry (expected mid-2025), switching requires a single .env change:
bash# Current
OLLAMA_MODEL=qwen2.5:3b

# Future — zero code change needed
ollama pull gemma4:latest
OLLAMA_MODEL=gemma4:latest
The OllamaClient created by the prompt above is model-agnostic by design. Your router in llm/router.py will automatically use whichever model OLLAMA_MODEL points to, with the Gemini path as a cloud fallback when Ollama is unreachable.
The three integrations complement each other precisely: code-review-graph reduces the tokens Antigravity consumes reading your codebase, caveman reduces the tokens your internal agent prompts consume at runtime, and vector-less RAG eliminates the retrieval infrastructure entirely by leveraging the local model's context window directly.
